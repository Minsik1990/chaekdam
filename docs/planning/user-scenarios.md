## 시나리오 1: 첫 독서 기록 작성 (신규 사용자)

### 사용자 정보

- **이름**: 김민지 (29세, 직장인)
- **독서 습관**: 월 2~3권, 감상 기록 경험 없음
- **동기**: 읽은 책의 인상 깊은 부분을 기록하고 싶음
- **기기**: iPhone 14 (모바일 퍼스트)

### 상황

출근길 지하철에서 "미드나잇 라이브러리" 완독. 인상 깊은 구절을 기록하고 싶다. 친구에게 독독 초대 코드를 받았다.

### 사용자 여정 (Step-by-Step)

```
1. 초대 코드 입력 (/invite)
   → "dokdok2026" 입력 → 유효 확인

2. 이메일 입력 (/login)
   → minji@email.com 입력 → "매직 링크를 보냈어요" 확인

3. 이메일 확인 → Magic Link 클릭
   → 자동 로그인 → /auth/callback → /onboarding

4. 닉네임 설정 (/onboarding)
   → "민지" 입력 → profiles 테이블 생성 → / (홈)

5. 홈 화면 (/)
   → 빈 상태 UI: "첫 기록을 남겨보세요"
   → 하단 "+" 버튼 클릭

6. 기록 작성 (/record/new)
   → 도서 검색: "미드나잇 라이브러리" → 결과에서 선택
   → 상태: "완독" 선택
   → 별점: 4점 (크래프트베이지 별)
   → 인용구: "후회는 인생의 재료일 뿐이다"
   → 감상: "평범한 일상이 얼마나 소중한지 깨닫게 해줬다"
   → 카드 배경색: 민트 선택
   → "저장" 클릭

7. 홈 (/)
   → 파스텔 민트 배경의 기록 카드 표시
   → "완독" 뱃지 + 별점 4점 표시
```

### 핵심 인사이트

- 기록 작성 3분 이내 완료 가능해야 함
- 빈 상태 UI가 행동을 유도해야 함
- 첫 기록 완료 후 시각적 성취감 (기록 카드)

### 관련 인수 기준

- AC-1 (인증), AC-2 (도서 검색), AC-3 (기록 CRUD), AC-4 (기록 카드 UI)

## 시나리오 2: 대화형 기록 (AI 인터뷰)

### 사용자 정보

- **이름**: 박준형 (34세, 스타트업 기획자)
- **독서 습관**: 주 1권, 감상 정리에 어려움
- **동기**: AI가 질문해주면 생각 정리가 쉬울 것 같음
- **기기**: Galaxy S24 (모바일)

### 상황

"아주 작은 습관의 힘" 완독. 실천 계획을 정리하고 싶지만, 빈 화면에 글쓰기가 막막하다.

### 사용자 여정

```
1. 기록 작성 화면 (/record/new)
   → 책 검색: "아주 작은 습관의 힘" 선택
   → "AI와 대화하며 기록하기" 버튼 클릭

2. AI 인터뷰 시작 (InterviewChat)
   AI: "이 책에서 가장 인상 깊었던 습관 전략은 무엇인가요?"
   준형: "2분 법칙입니다. 무엇이든 2분만 하면 습관이 된다는 거요."

   AI: "2분 법칙을 준형님의 일상에 적용한다면 어떤 습관을 만들고 싶으신가요?"
   준형: "아침에 일어나서 책 2분만이라도 읽으려고요."

   AI: "좋은 계획이에요. 이 책의 핵심 메시지를 한 문장으로 정리한다면?"
   준형: "아주 작은 시작이 큰 변화를 만든다."

3. 대화 완료 → 요약 생성 (/api/agent/summarize)
   → AI가 대화 내용을 감상문 형태로 정리
   → 기록 폼에 자동 입력

4. 수정 → 저장
   → 준형이 요약본 약간 수정 → "저장" 클릭
   → 홈에서 기록 카드 확인
```

### 핵심 인사이트

- AI는 3~5개 질문만 (너무 길면 이탈)
- 자연스러운 대화 흐름 (Haiku 4.5 사용)
- 요약본 자동 생성 + 사용자 수정 가능
- 3턴 이상 대화 후 "정리하기" 버튼 표시

### 관련 인수 기준

- AC-5 (대화형 기록), AC-6 (AI 에이전트)

## 시나리오 3: 독서 모임 관리

### 사용자 정보

- **이름**: 이서연 (31세, 마케터)
- **독서 습관**: 월 1~2권, 독서 모임 운영 2년차
- **동기**: 모임 기록을 체계적으로 관리하고 싶음
- **기기**: MacBook (데스크톱)

### 상황

"목요 독서회" 모임을 운영 중. 이번 달 모임에서 "사피엔스"를 읽기로 했다.

### 사용자 여정

```
1. 모임 만들기 (/groups/new)
   → "모임 만들기" 탭 선택
   → 모임명: "목요 독서회"
   → 설명: "매월 셋째 주 목요일 저녁 7시"
   → "만들기" 클릭
   → 초대 코드 생성 (예: "thu-book-42")
   → 초대 코드를 카카오톡에 공유

2. 멤버 가입
   → 모임원 3명이 초대 코드로 가입
   → 모임 상세에서 멤버 목록 확인

3. 세션 추가 (/groups/[id]/sessions/new)
   → 날짜: 2026-02-20
   → 책 검색: "사피엔스" → 선택
   → "세션 추가" 클릭

4. 세션 상세 (/groups/[id]/sessions/[sessionId])
   → AI 패널에서 "토론 주제" 탭 클릭
   → AI가 토론 주제 5개 생성 (SSE 스트리밍)
   → "발제문 초안" 탭에서 발제문 자동 생성

5. 모임 후 → 감상 작성
   → 서연: "호모 사피엔스의 인지혁명이 가장 흥미로웠다"
   → 별점 5점 → "감상 남기기"
   → 다른 멤버들도 감상 작성
```

### 핵심 인사이트

- 모임 초대 코드 공유가 쉬워야 함
- AI 토론 주제가 실제 모임에 유용해야 함
- 세션별 감상 기록으로 모임 히스토리 축적

### 관련 인수 기준

- AC-7 (독서 모임), AC-8 (세션 후기), AC-6 (AI 에이전트)

## 시나리오 4: AI 도서 분석 활용

### 사용자 정보

- **이름**: 정하준 (27세, 대학원생)
- **독서 습관**: 주 2~3권, 심층 분석 선호
- **동기**: 책의 핵심 내용을 빠르게 파악하고 싶음
- **기기**: iPad (태블릿)

### 상황

"총균쇠"를 읽기 시작했는데, 방대한 분량에 핵심 파악이 어렵다.

### 사용자 여정

```
1. 기록 작성 (/record/new)
   → "총균쇠" 검색 → 선택
   → 상태: "읽는 중"
   → 간단한 메모만 남기고 저장

2. 기록 상세 (/record/[id])
   → 기록 카드 하단에 AI 분석 버튼

3. AI 분석 패널 (AgentPanel)
   → "분석" 탭 클릭
   → /api/agent/analysis 호출 → 캐싱 확인
   → 없으면 Sonnet 4.5로 분석 생성:
     - 핵심 논점 3가지
     - 챕터별 요약
     - 관련 도서 추천

4. 토론 주제 활용
   → "토론 주제" 탭에서 5개 주제 확인
   → 독서 모임 세션에서 활용
```

### 핵심 인사이트

- AI 분석 결과는 DB 캐싱 (ai_contents)
- 동일 책 재요청 시 즉시 반환 (비용 절감)
- 분석 품질이 사용자 만족도에 직결

### 관련 인수 기준

- AC-6 (AI 에이전트), AC-12 (성능)

## 시나리오 5: 재방문 사용자 (일상적 사용)

### 사용자 정보

- **이름**: 김민지 (시나리오 1과 동일)
- **상황**: 가입 2주 후, 기록 5개 작성

### 사용자 여정

```
1. 앱 접속 (/)
   → 자동 로그인 (세션 유지)
   → 홈에서 최근 기록 5개 카드 타임라인

2. 프로필 확인 (/profile)
   → 독서 통계: 완독 3권, 읽는 중 1권, 읽고 싶은 1권
   → 평균 별점: 4.2점

3. 기존 기록 수정 (/record/[id]/edit)
   → "미드나잇 라이브러리" 기록 클릭
   → 감상 추가 작성 → 저장

4. 새 책 기록 추가
   → 하단 "+" 버튼 → 빠른 기록 작성
```

### 핵심 인사이트

- 재방문 시 빠른 로딩 (LCP < 3초)
- 기존 기록 수정이 자연스러워야 함
- 통계가 동기 부여 역할
