# 기능별 스펙 문서 (Feature Specs)

> 큰 작업 (새 기능, 전면 개편) 시 기획서를 여기에 작성합니다.
> 작은 작업 (버그 수정, UI 미세 조정)은 이 문서 불필요.

## 폴더 구조

```
docs/features/
├── README.md              # 이 파일
├── {기능명}/
│   ├── v1-spec.md         # 초기 기획 (수정하지 않음)
│   ├── v2-update.md       # 기능 변경/확장 시 추가
│   └── v3-overhaul.md     # 전면 개편 시 추가
```

## 규칙

1. **이전 버전 파일은 수정하지 않음** → 새 버전 파일 추가
2. 파일명: `v{번호}-{설명}.md`
3. 기획서에는 반드시 포함: 배경/목적, 요구사항, 기술 설계, 구현 계획
